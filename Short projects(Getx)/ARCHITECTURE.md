# ARCHITECTURE.md — GetX Boilerplate
> Small-to-medium projects. Single developer or small team. Feature count ≤ 5–6. No complex background sync.
> Stack: GetX · GetIt + Injectable · Retrofit + Dio · GetStorage / Hive · Flutter Gen · EasyLoading

## Entry Point

```dart
// main.dart
void main() => configuration(myApp: const MyApp());
```

`configuration()` lives in `utils/helpers/injectable/injectable.dart`. It:
1. Calls `WidgetsFlutterBinding.ensureInitialized()`
2. Runs `getIt.init()` (Injectable generated)
3. *(Optional)* Boots Firebase Crashlytics — only when Firebase is part of the project
4. Configures `EasyLoading`
5. Calls `runApp(myApp)`

In release mode it wraps everything in `runZonedGuarded` → Firebase Crashlytics fatal error handler.
In debug mode it runs directly (no zone).

> **Firebase is optional.** Add it only when the project explicitly requires it. If absent, remove Crashlytics lines and `firebase_options.dart`.

```dart
// injectable.dart — signature
@i.injectableInit
void configuration({required Widget myApp}) { ... }

final getIt = GetIt.instance;
```

## Dependency Injection — GetIt + Injectable

All dependencies registered via `@injectable` / `@lazySingleton` annotations.
Code generated by `build_runner` → `injectable.config.dart`.

```dart
// RegisterModule — injectable_properties.dart
@module
abstract class RegisterModule {
  @singleton
  Dio dio() => Dio(BaseOptions(baseUrl: AppConfig.baseUrl))
    ..interceptors.addAll([AppDioInterceptor(), if (kDebugMode) PrettyDioLogger()]);

  @preResolve
  Future<GetStorage> storage() async {
    await GetStorage.init();
    return GetStorage();
  }
}
```

Rules:
- Controllers → `@lazySingleton`
- Services (Retrofit) → `@lazySingleton` with `@factoryMethod`
- One-time async setup (storage, prefs) → `@preResolve`
- Never call `getIt<T>()` inside `build()` — inject via constructor or access through `GetItHook`

## Routing — GetX

```dart
// routes/app_routes.dart
class AppRoutes {
  static const splash   = '/splash';
  static const home     = '/home';
  // Add per project. Use /featureName convention.
}

// routes/app_pages.dart
class AppPages {
  static final routes = [
    GetPage(name: AppRoutes.splash, page: () => const SplashPage()),
    GetPage(name: AppRoutes.home,   page: () => const HomePage()),
  ];
}
```

Navigation:
```dart
Get.toNamed(AppRoutes.home);
Get.offAllNamed(AppRoutes.splash);
Get.back();
```

Never use `Navigator.push` directly.

## Controller Lifecycle — GetItHook

Every page widget extends `GetItHook<T>` instead of `StatelessWidget`.

```dart
abstract class GetItHook<T extends GetxController> extends StatefulWidget {
  const GetItHook({super.key});
  bool get autoDispose;
  Widget build(BuildContext context);
  T get controller => getIt<T>();

  @override
  State<GetItHook> createState() => _GetItHookState<T>();
}

class _GetItHookState<T extends GetxController> extends State<GetItHook> {
  @override
  Widget build(BuildContext context) => widget.build(context);

  @override
  void dispose() {
    super.dispose();
    if (widget.autoDispose && getIt.isRegistered<T>()) {
      getIt.resetLazySingleton<T>();
    }
  }
}
```

Usage:
```dart
class HomePage extends GetItHook<HomeController> {
  const HomePage({super.key});

  @override
  bool get autoDispose => true;   // false for root/persistent controllers

  @override
  Widget build(BuildContext context) {
    return AppScaffold(
      title: context.T.homeTitle,
      body: (context) => ...,
    );
  }
}
```

- `autoDispose: true` → resets the lazySingleton on page pop (screen-scoped state)
- `autoDispose: false` → keeps controller alive (root pages, bottom nav tabs)

## Theme System

Three-layer system. Never bypass any layer.

### Layer 1 — KColors (raw constants)
```dart
// Never use directly in widgets. Used only as defaults in AppColors.
KColors.primary   // Color(0xFF...)
KColors.white
KColors.bg1
```

### Layer 2 — AppColors (semantic, context-aware)
```dart
final colors = AppColors.of(context);
colors.primary     // adapts to theme
colors.bg1
colors.fieldColor
```

### Layer 3 — AppStyles (typography)
```dart
final style = AppStyles.of(context);
style.s16w400Black
style.s20w600Black
```

`AppStyles` uses the `Poppins` class (extends `TextStyle`) — never instantiate `TextStyle(...)` directly.

```dart
class Poppins extends TextStyle {
  const Poppins({super.fontSize, super.color, super.fontWeight, ...})
      : super(fontFamily: 'Poppins');
}
```

Both are `ThemeExtension`s registered in `AppTheme.lightTheme`. Dark theme follows the same structure — add a `darkTheme` with overridden `AppColors` values.

## Localization

ARB files in `l10n/`. Generated class: `AppLocalizations`.

Access strings **only** via:
```dart
context.T.someKey       // inside widgets (BuildContext available)
AppStrings.T.someKey    // inside controllers (uses Get.context!)
```

Never hardcode user-facing strings.

```dart
// context extension
extension BuildContextX on BuildContext {
  AppLocalizations get T => AppLocalizations.of(this)!;
}

// AppStrings — for use outside widget tree
class AppStrings {
  AppStrings._();
  static AppLocalizations get T => AppLocalizations.of(Get.context!)!;
}
```

Locale is managed by `LocaleController` (`@lazySingleton`) — reads/writes to `GetStorage`.

## API Layer — Retrofit + Dio

```dart
@lazySingleton
@RestApi()
abstract class AuthService {
  @factoryMethod
  factory AuthService(Dio dio) = _AuthService;

  @POST(EndPoints.login)
  Future<LoginResponse> login(@Body() LoginRequest request);
}
```

- Base URL set in `RegisterModule` on the `Dio` instance
- `AppDioInterceptor` handles auth headers + token refresh
- `PrettyDioLogger` in debug only
- Models use `freezed` + `json_serializable`

## Local Storage

- **GetStorage** — lightweight key-value (locale, token, simple flags)
- **Hive** — typed boxes for larger/structured local data

Access GetStorage via DI: `getIt<GetStorage>()`. Never create a new instance directly.
